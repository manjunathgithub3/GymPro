# Payment Use Cases

Version: 1.0
Status: Draft
Project: GymPro

## Overview
This document defines payment management use cases for GymPro.

## Primary Actors
- Staff/Admin
- Member
- System

## Use Cases

### UC-PAY-001: Record Payment
- Description: Staff records a payment received from a member.

### UC-PAY-002: Partial Payment
- Description: Member pays part of an outstanding dues amount.

### UC-PAY-003: View Payment History
- Description: User views all payment transactions.

### UC-PAY-004: Generate Invoice
- Description: System generates an invoice for payment or membership purchase.
